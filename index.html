3 Определить размер файла file.tar.gz, не скачивая его ✅ 1,048,576 байт

▼

📖 Что нужно сделать?

Использовать метод HEAD вместо GET. HEAD запрашивает только заголовки ответа без тела, что позволяет получить метаинформацию о файле без его загрузки.

🔧 Отправленный запрос:
HEAD /task2/files/file.tar.gz HTTP/1.1
Host: u82591.kubsu-dev.ru
Connection: close

(пустая строка в конце)

📌 Полученный результат:

  • HTTP/1.1 200 OK — файл существует
  • Content-Length: 1048576 — размер файла (1 МБ = 1,048,576 байт)
  • Content-Type: application/x-gzip — тип файла
  • Тело ответа отсутствует (файл не скачан)

📸 Скриншот выполнения:

4 Определить медиатип ресурса /image.png ✅ image/png

▼

📖 Что нужно сделать?

Использовать метод HEAD для получения только заголовков. Медиатип (MIME-type) определяется из заголовка Content-Type.

🔧 Отправленный запрос:

Host: u82591.kubsu-dev.ru

(пустая строка в конце)

📌 Полученный результат:

  • HTTP/1.1 200 OK — файл существует
  • Content-Type: image/png — медиатип PNG изображения
  • Content-Length: 11 — размер файла (тестовый)

5 Отправить комментарий на сервер по адресу /index.php ✅ POST успешен

▼

📖 Что нужно сделать?

Отправить POST-запрос с данными комментария. POST передает данные в теле запроса, что требует указания заголовков Content-Type и Content-Length.

🔧 Отправленный запрос:

Host: u82591.kubsu-dev.ru
Content-Type: application/x-www-form-urlencoded
Content-Length: 21
Connection: close

comment=test_comment

📌 Полученный результат:

  • HTTP/1.1 200 OK — запрос успешно обработан
  • Content-Length: 52 — размер ответа
  • Сервер обработал комментарий (ответ должен содержать "Получен комментарий")

📸 Скриншот выполнения:

6 Получить первые 100 байт файла /file.tar.gz ✅ 206 Partial Content

▼

📖 Что нужно сделать?

Использовать заголовок Range для частичной загрузки файла. Это позволяет получить только указанный диапазон байт, не скачивая весь файл.

🔧 Отправленный запрос:
GET /task2/files/file.tar.gz HTTP/1.1
Host: u82591.kubsu-dev.ru
Range: bytes=0-99
Connection: close

(пустая строка в конце)

📌 Полученный результат:

  • HTTP/1.1 206 Partial Content — частичное содержимое
  • Content-Range: bytes 0-99/1048576 — запрошены байты 0-99 из 1,048,576
  • Content-Length: 100 — получено 100 байт
  • Тело ответа содержит первые 100 байт файла

📸 Скриншот выполнения:

📖 Что нужно сделать?

  • Это стандартная кодировка для веб-страниц, поддерживающая все символы Юникода
